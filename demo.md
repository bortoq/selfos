# Demo

## Scenario: Running Life As A Project

A user opens Self OS at the start of a new two-week sprint.

## Sprint Start

The user defines:

- one main goal;
- three key outcomes;
- a few metrics to watch, such as energy, focus, and overload.

## Daily Loop

In the morning, the system asks a short check-in.

In the evening, the system asks:

- what was completed;
- how the user felt;
- what blocked progress;
- whether anything should change tomorrow.

The user can also add a private note.

## Mid-Sprint Insight

After several days, the system combines manual check-ins with imported data and highlights a pattern, such as lower productivity on poor-sleep days or repeated slippage on communication-heavy tasks.

## Retrospective

At the end of the sprint, the AI advisor drafts a retrospective:

- what improved;
- what stayed flat;
- what likely caused missed outcomes;
- what to change in the next sprint.

## Public Profile

If the user enables sharing, the system generates a static public profile with selected progress metrics, the current sprint, recent achievements, and high-level trends.

## Demo Goal

The first demo should prove a full personal operating loop: short check-ins, measurable sprint progress, reflection, and a generated profile from the same underlying data.
